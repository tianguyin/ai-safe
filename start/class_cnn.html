典型架构卷积神经网络的参数恢复#

为什么要进行参数恢复#

对于典型架构卷积神经网络，我们必须要获取其cnn参数才能正常运行其权重文件，这是后续攻击的基础。

class cnn就像是一个接口，告诉框架该模型怎么去调用。

获取CNN（模型类名）#

例题 2025 软件安全赛（天津赛区）ez_sight ai部分 这里采用这道题目，是因为这是一个已经训练好了的典型架构卷积神经网络的权重模型（我懒的自己训练一个了），本篇文章也是建立在典型架构卷积神经网络的基础上进行参数恢复

当然，当时这道题目给了一个公告文档 不过我这里并不打算通过这个文档解题，这里我们通过恢复参数的方式进行解题

class CNN(nn.Module):

那么，我们可以通过 构造一个无用的SimpleCNN 来尝试调用该权重文件

如果成功，就可以通过该方法获取SimpleCNN的模型参数

import torch
import torch.nn as nn


class SimpleCNN(nn.Module):
    def __init__(self):
        super(SimpleCNN, self).__init__()
        pass

    def forward(self, x):
        pass


# 加载模型
model = torch.load('password.pt', map_location='cpu', weights_only=False)


def analyze_cnn_structure(model):
    print(f"模型类名: {model.__class__.__name__}")
    print()

    # 分析每一层，按类型分类
    conv_layers = []
    fc_layers = []
    other_layers = []

    # 遍历模型的所有模块
    for name, module in model.named_modules():
        if isinstance(module, nn.Conv2d):
            conv_layers.append((name, module))
        elif isinstance(module, nn.Linear):
            fc_layers.append((name, module))
        elif isinstance(module, (nn.BatchNorm2d, nn.ReLU, nn.MaxPool2d, nn.Dropout)):
            other_layers.append((name, module))

    # 打印卷积层信息
    print("卷积层:")
    for name, layer in conv_layers:
        print(f"  {name}: {layer}")
        print(f"    输入通道数: {layer.in_channels}")
        print(f"    输出通道数: {layer.out_channels}")
        print(f"    卷积核大小: {layer.kernel_size}")
        print(f"    步长: {layer.stride}")
        print(f"    填充: {layer.padding}")
        print()

    # 打印全连接层信息
    print("全连接层:")
    for name, layer in fc_layers:
        print(f"  {name}: {layer}")
        print(f"    输入特征数: {layer.in_features}")
        print(f"    输出特征数: {layer.out_features}")
        print()

    # 打印其他层信息
    print("其他层:")
    for name, layer in other_layers:
        print(f"  {name}: {layer}")

    # 显示参数统计
    print("\n参数统计:")
    total_params = 0
    for name, param in model.named_parameters():
        param_count = param.numel()
        total_params += param_count
        print(f"  {name}: {param.shape} -> {param_count} 个参数")

# 调用分析函数
analyze_cnn_structure(model)

推断数据流动路径#

def forward(self, x): x = self.pool(F.relu(self.conv1(x))) x = self.pool(F.relu(self.conv2(x))) x = x.view(x.size(0), -1) x = F.relu(self.fc1(x)) x = self.fc2(x) return x

基于组件信息，可以按以下步骤重构 forward 逻辑：

（1）卷积部分#

  • 典型 CNN 的卷积部分遵循 卷积 → 激活 → 池化 的顺序。

  • 每一层的输出通道数需与下一层的输入通道数匹配。

    self.conv1(x)：对输入进行第一次卷积操作。

    F.relu(...)：应用 ReLU 激活函数引入非线性。

    self.pool(...)：使用最大池化降维，保留主要特征。

  • 重复上述步骤完成第二个卷积块。

（2）展平操作#

  • 卷积输出需要通过 view() 或 flatten() 转换为一维向量，才能输入全连接层。

  • 展平后的维度需与 fc1 的输入特征数一致（本例中为 32 * 8 * 8）。

    保留批次维度：x.size(0) 表示批次大小（batch size），即当前批次中有多少个样本。

    展平特征维度：-1 是一个特殊参数，表示 "自动计算该维度的大小"，PyTorch 会根据张量的总元素数和已知维度（x.size(0)）来推断这个维度的值。

（3）全连接部分#

  • 通常包含非线性激活（如 ReLU），但最后一层分类任务一般不加激活（直接接 softmax）。

    x = F.relu(self.fc1(x)) # 第一个全连接层 + ReLU 激活 x = self.fc2(x) # 第二个全连接层（输出层，无激活）

现在我们来看公告中的用例守则

请注意公司内部AI模型的使用规范： 1.除最后一层外与池化层外其他隐藏层输出均需要通过激活函数 2.至少需要通过两次池化层 3.注意隐藏之间输出数据格式的匹配，必要时对数据张量进行重塑 4.为保证模型准确性，输入图片应转换为灰度图
